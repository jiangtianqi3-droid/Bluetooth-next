# Changelog

## v1.2.0 - 2026-05-15

### 版本信息

- `versionCode = 3`
- `versionName = 1.2.0`
- Debug APK：`app/build/outputs/apk/debug/app-debug.apk`
- Release APK：`app/build/outputs/apk/release/app-release.apk`

### 新增了什么

- 新增目标提醒设置：
  - 单次最长使用提醒，默认 `2小时`。
  - 连续佩戴休息提醒，默认 `60分钟`。
  - 每周目标提醒，默认 `每日额度 × 7`。
  - 睡前使用提醒，默认关闭，开启后只提醒不停止记录。
- 顶部设置按钮旁新增“今天主要听了什么”入口，按音乐、视频、播客/音频、其他分类展示今日播放来源统计，并显示具体 App 时长。
- 今日条带增加轻量小时刻度和当前时间线，帮助快速判断现在处于清醒时间轴中的位置。

### 修复了什么

- 发布流程开始固定记录 APK 路径、版本号、更新内容和旧数据清理说明。
- 新提醒全部使用可清除的普通提醒通知，不影响连接中的主监控通知。

### 是否需要清理旧数据

- 不需要强制清理旧数据。
- 本次不升级 Room 数据库，不改历史记录结构。
- 若之前存在非目标设备、超短噪声或重复记录，仍可在设置页使用“清理异常记录”手动处理。

## v1.1.0 - 2026-05-14

### 更新说明

- 后台省电优化：
  - 移除了 `MainViewModel` 中 10 秒一次的蓝牙连接轮询，避免前台页面重复探测后台状态。
  - 将服务侧固定 30 秒循环拆分为低频连接兜底检查和仅活跃会话期间运行的额度检查。
  - 调整后台恢复心跳策略：只在开启开机自启动且已配置目标设备时保留，周期放宽到 60 分钟。
  - 蓝牙 profile 查询改为缓存复用，减少重复 `getProfileProxy()` 带来的系统开销。
  - 电量读取改为“连接时读取 + 广播更新 + 未知电量低频兜底”，减少无意义的周期查询。
  - 媒体通知监听新增去抖和内容去重，降低频繁写入 DataStore 与通知刷新的开销。

- 稳定性改进：
  - 空闲无连接状态下，服务更积极自停，避免后台空转。
  - 通知刷新增加内容去重，未变化时不再重复通知系统 UI。
  - 开机恢复、蓝牙广播和活跃会话恢复路径进一步统一。

### 适用说明

- 这是一个可回滚版本，仓库会同步打上 `v1.1.0` 标签。
- Android 构建版本更新为：
  - `versionCode = 2`
  - `versionName = 1.1.0`
